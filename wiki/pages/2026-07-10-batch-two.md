## Tracker (tracker/)

- evdev input counting on linux (tracker/src/input_evdev.rs): background thread
  counts keydowns and clicks into shared atomics, rescans devices every 30s, on
  EACCES logs once and retries every 60s so it self-heals when the udev rule
  lands. Hyprland backend consumes per-tick deltas; X11 keeps device_query.
- Idle fix (tracker/src/idle_tracking.rs): window TITLE change now counts as
  activity in the no-keys fallback branch. This fixes the Wayland keyboard-only
  undercount (nvim work marked idle). Threshold unchanged at 5min.
- Hyprland event socket (tracker/src/hypr_events.rs) replaces per-tick hyprctl
  subprocess polling: subscribes to .socket2.sock activewindow events with
  reconnect backoff, 5s reconciliation via hyprctl activewindow -j, PID resolved
  once per focus change. Real bug found during verification: event-socket
  addresses omit the 0x prefix that hyprctl JSON includes; addresses are
  normalized (unit-tested) to prevent spurious span splits.
- tmux drill-down: on terminal focus, subProgram resolves from a push file
  (~/.local/state/chronomaxi/foreground, written by zsh preexec/precmd and tmux
  hooks from deploy/drilldown/, fresh under 10s) else tmux IPC
  (pane_current_command), else none. Terminal windows use effective identity
  windowId:subProgram for span-change detection so nvim vs zsh split spans.
  Hooks installed on big-ron.
- Categories (tracker/src/category.rs): alacritty and kitty map to Coding;
  subProgram overrides (nvim, git, omp, claude, cargo, bun, ...) win before set
  lookups.

## Incidents and the ops rule they bought

1. A subagent kill-tested a brave --app pid; Brave is single-instance, so it
   SIGTERMed the user's whole browser (all tabs; recovered via session
   restore).
2. A subagent ran pkill -f "target/release/backend" to clean a smoketest and
   took down the production tracker on big-ron (systemd restarted it).
3. The orchestrator killed a convex dev process on port 3210 before verifying
   its parent; it belonged to an unrelated project (starcube-autoresearch).

Rule, now standing: verify process identity (cwd, owner, parentage, purpose)
BEFORE any kill. Never signal browser processes. Never pkill by pattern, exact
PID only.
